# AIYOU 漫剧创作平台新手入门指南

## 一、项目简介

AIYOU 是一款革命性的 AI 驱动漫剧创作平台，让创作者通过简单的节点式操作，完成从创意构思到最终视频成片的全流程制作。本项目采用纯 AI 开发模式，从创意到实现全程借助 Claude、Cursor、ChatGPT 等 AI 工具完成代码编写，体现了 AI 技术在软件开发领域的巨大潜力。

AIYOU 的核心理念是「一个人就是一个团队」，通过集成多个顶级 AI 模型，提供从剧本创作、角色设计、分镜制作到视频生成的全流程解决方案。平台支持多种 AI 模型接入，包括 Gemini、云屋 API、Sora2、Kling、Luma、Runway 等，创作者可以根据需求灵活选择和切换模型，实现最优的创作效果。整个系统采用模块化设计，各节点之间通过标准化的数据接口进行通信，确保了系统的可扩展性和可维护性。

本指南旨在帮助新手用户快速上手 AIYOU 平台，从环境搭建到实际创作，涵盖所有必要的操作步骤和注意事项。无论您是技术背景还是非技术背景的用户，都能通过本指南顺利开始您的漫剧创作之旅。

## 二、系统要求

### 2.1 硬件要求

在部署 AIYOU 平台之前，首先需要确保您的开发环境满足以下硬件要求。这些要求基于实际测试和用户反馈得出，能够保证平台运行的流畅性和稳定性。硬件配置直接影响平台的响应速度和用户体验，因此建议在条件允许的情况下，尽可能满足推荐配置而非最低配置。

最低硬件配置要求包括：处理器需要 Intel Core i5 或同等性能的 AMD 处理器，内存至少需要 8GB RAM，硬盘空间需要至少 20GB 可用存储，网络带宽建议不低于 10Mbps。这样的配置能够支持基本的创作流程，但在处理复杂任务时可能会出现响应延迟的情况。对于日常使用和轻度创作，最低配置已经能够满足基本需求。

推荐硬件配置要求包括：处理器升级为 Intel Core i7 或 AMD Ryzen 7 以上级别，内存提升至 16GB RAM，硬盘建议使用 SSD 固态硬盘以提升读写速度，网络带宽建议不低于 50Mbps 以确保 AI 接口调用的稳定性。推荐配置能够提供更流畅的用户体验，特别是在处理高清视频生成和多节点联动等复杂场景时表现更加出色。

专业级硬件配置适用于需要进行大规模创作或有高并发需求的用户：处理器建议使用 Intel Core i9 或 AMD Ryzen 9，内存提升至 32GB 或更高，使用高速 NVMe SSD 硬盘，网络带宽建议不低于 100Mbps。专业配置不仅能够提升单个用户的创作效率，还能够支持多用户协作场景。

### 2.2 软件要求

软件环境是 AIYOU 平台运行的基础，正确的软件版本和配置对于平台的稳定运行至关重要。以下是各个核心软件的版本要求和建议配置，涵盖操作系统、运行时环境和开发工具等方面。

操作系统方面，AIYOU 支持 macOS、Windows 和 Linux 三大主流操作系统。macOS 用户建议使用 macOS 12 Monterey 或更高版本，Windows 用户建议使用 Windows 10 21H2 或 Windows 11，Linux 用户建议使用 Ubuntu 20.04 LTS 或同等水平的发行版。不同操作系统在界面表现和部分功能上可能存在细微差异，但核心功能在所有系统上保持一致。

Node.js 环境是后端服务运行的必要条件，AIYOU 要求 Node.js 版本不低于 18.0.0，建议使用 LTS（长期支持）版本以获得最佳的稳定性和安全性。您可以通过访问 Node.js 官方网站（https://nodejs.org）下载安装包，或使用 nvm（Node Version Manager）进行版本管理。安装完成后，可以通过在终端执行 `node -v` 命令来验证 Node.js 是否正确安装。

包管理器方面，平台支持 npm、yarn 和 pnpm 三种主流的 Node.js 包管理器。推荐使用 pnpm 作为首选包管理器，它具有更快的安装速度和更高效的磁盘空间利用率。如果选择使用 npm，建议版本不低于 9.0.0；如果选择 yarn，建议版本不低于 3.0.0。不同包管理器在依赖解析策略上可能存在差异，因此建议在整个项目中统一使用同一种包管理器。

Git 版本控制工具是协作开发和代码管理的必备工具，建议 Git 版本不低于 2.30.0。Git 不仅用于代码版本管理，还是获取项目更新和提交问题反馈的重要渠道。如果您计划参与 AIYOU 的开发工作，还需要配置 SSH 密钥以连接到 GitHub 等代码托管平台。

### 2.3 浏览器要求

AIYOU 的前端界面运行在现代浏览器环境中，为了获得最佳的用户体验，建议使用 Chrome、Edge 或 Safari 等主流浏览器的最新稳定版本。浏览器需要支持 ES2020 模块系统、WebGL 渲染和 Canvas API 等技术特性，这些特性在主流浏览器中均已得到良好支持。

推荐使用的浏览器版本包括：Chrome 100 以上版本、Edge 100 以上版本、Safari 15 以上版本或 Firefox 100 以上版本。强烈建议启用浏览器的硬件加速功能，以提升 Canvas 渲染和视频处理的性能。在浏览器设置中，确保已开启「在可用时使用硬件加速」选项。

对于国内用户，由于网络环境的原因，Chrome 浏览器可能需要配置代理或使用国内镜像才能正常访问。如果在使用过程中遇到页面加载缓慢或部分资源无法加载的问题，建议检查网络连接或尝试更换浏览器。平台的前端代码已经针对国内网络环境进行了优化，CSS 和 JavaScript 资源优先从国内 CDN 加载。

## 三、项目部署

### 3.1 获取项目代码

获取 AIYOU 项目代码是部署的第一步，您可以通过 Git 克隆或下载压缩包的方式获取项目源代码。无论采用哪种方式，都需要确保网络连接稳定，以便完整下载所有项目文件和历史记录。推荐使用 Git 克隆的方式获取项目，这样可以在后续方便地获取更新和参与代码贡献。

使用 Git 克隆项目的具体步骤如下：首先打开终端（macOS/Linux 用户使用 Terminal，Windows 用户使用 PowerShell 或命令提示符），然后执行克隆命令。如果您的网络环境支持 Git 协议，可以直接使用 SSH 方式克隆，这样不需要每次都输入用户名和密码。克隆完成后，项目文件将保存在当前目录下的「哎呦漫剧生成」文件夹中。

如果您的网络环境不支持 Git 协议，可以尝试使用 HTTPS 方式克隆，或通过 GitHub 网页下载项目的 ZIP 压缩包。下载完成后，需要将压缩包解压到目标目录。解压后，项目结构应该与通过 Git 克隆得到的结构完全一致。请确保解压后的路径中不包含中文字符或特殊符号，以避免某些工具在处理路径时出现兼容性问题。

### 3.2 前端部署

前端是用户与平台交互的主要界面，部署前端需要安装依赖、配置环境变量和启动开发服务器三个主要步骤。整个过程相对简单，但需要注意依赖版本的兼容性和环境变量的正确配置。以下是前端部署的详细步骤，请按照顺序逐步执行。

第一步是进入前端项目目录并安装依赖。打开终端，使用 cd 命令进入项目根目录，然后执行依赖安装命令。使用 pnpm 安装依赖时，只需要执行 `pnpm install`；如果使用 npm，则执行 `npm install`；如果使用 yarn，则执行 `yarn install`。安装过程中会自动下载项目所需的全部依赖包，这个过程可能需要几分钟时间，具体取决于网络速度和计算机性能。

```bash
# 进入项目目录
cd /Volumes/光波/哎呦漫剧生成

# 安装前端依赖
pnpm install
```

依赖安装完成后，需要配置环境变量文件。前端项目使用 `.env.local` 文件存储敏感配置，如 API Keys 等。在项目根目录下，应该已经存在一个 `.env.local` 文件，您需要用文本编辑器打开该文件，并根据文件中的提示填入相应的 API Key。如果您使用 Git 克隆项目后发现该文件不存在，可以从 `.env.example` 文件复制并重命名来创建。

环境变量配置完成后，就可以启动前端开发服务器了。执行 `pnpm dev` 命令启动开发服务器，首次启动时 Vite 会进行一些初始化工作。启动成功后，您应该能够在终端看到类似「Local: http://localhost:5173/」的提示信息。这意味着前端服务已经成功运行，打开浏览器访问该地址即可看到 AIYOU 的界面。

```bash
# 启动前端开发服务器
pnpm dev
```

如果一切正常，您现在应该能看到 AIYOU 的欢迎界面。如果您看到的是空白页面或错误信息，请检查浏览器的控制台（按 F12 打开开发者工具，切换到 Console 标签）中的错误提示，并根据错误信息进行排查。常见的启动问题包括端口被占用、依赖未正确安装、环境变量配置错误等。

### 3.3 后端部署

后端服务主要负责文件上传、API 代理和模型配置管理等功能，是平台正常运行的重要组成部分。虽然部分功能可以在纯前端模式下运行，但为了使用完整的平台能力，建议同时部署后端服务。后端部署的流程与前端类似，主要包括依赖安装、环境配置和服务启动三个步骤。

首先进入后端服务目录并安装依赖。后端服务位于项目根目录下的 `server` 文件夹中，这个目录有独立的 `package.json` 文件和 `node_modules` 依赖目录。执行依赖安装命令后，后端所需的 Express、CORS、Multer、COS SDK 等依赖包都会被下载安装到该目录中。

```bash
# 进入后端服务目录
cd server

# 安装后端依赖
pnpm install
```

后端服务需要配置腾讯云 COS（对象存储）相关的信息才能正常使用文件上传功能。在 `server` 目录下，应该存在 `.env.example` 文件，您需要将其复制为 `.env` 文件，并填入您的腾讯云 COS 配置信息。这些配置包括存储桶名称（Bucket）、地域（Region）、SecretId 和 SecretKey。获取这些信息需要您拥有腾讯云账号，并在腾讯云控制台中创建 COS 存储桶和访问密钥。

```bash
# 复制环境变量示例文件
cp .env.example .env
```

使用文本编辑器打开 `.env` 文件，按照文件中的提示填入您的配置信息。OSS_BUCKET 是您创建的 COS 存储桶名称，OSS_REGION 是存储桶所在的地域（如 ap-guangzhou 表示广州地域），OSS_SECRET_ID 和 OSS_SECRET_KEY 是您的腾讯云 API 访问密钥。请妥善保管您的 API 密钥，不要将包含真实密钥的配置文件提交到公开的代码仓库中。

配置完成后，可以启动后端服务进行测试。执行 `pnpm dev` 命令（开发模式，支持热重载），或执行 `pnpm start` 命令（生产模式）。开发模式下，当您修改后端代码时，服务会自动重启以应用更改，这对于开发和调试非常方便。服务启动后，默认监听 3001 端口。

```bash
# 开发模式启动（支持热重载）
pnpm dev

# 或生产模式启动
pnpm start
```

后端服务启动后，会在终端显示服务地址信息。您可以通过访问 `http://localhost:3001/api/health` 来检查服务是否正常运行，正常情况下应该返回包含健康状态信息的 JSON 响应。如果服务无法启动，请检查端口是否被占用、COS 配置是否正确、以及各项依赖是否完整安装。

### 3.4 数据库与存储配置

AIYOU 平台使用 IndexedDB 进行本地数据存储，这意味着您不需要部署额外的数据库服务。IndexedDB 是浏览器内置的 NoSQL 数据库系统，能够存储大量的结构化数据，包括项目配置、节点数据、生成的图片和视频等信息。所有数据都存储在用户的本地设备上，既保证了数据的安全性，又避免了服务器端数据库的部署复杂度。

对于文件存储，平台默认使用腾讯云 COS 作为对象存储服务。在上传图片和视频时，文件会首先上传到 COS 存储桶中，然后返回公开访问 URL。这种设计既解决了本地存储空间的限制，又提供了良好的访问性能。如果您没有配置 COS 服务，平台会使用内存临时存储，生成的视频只能临时访问，无法持久保存。

对于视频生成功能，平台支持对接多个视频生成服务，包括 Sora2、Kling、Luma 和 Runway 等。这些服务需要您单独申请 API Key 并在平台中进行配置。不同的视频生成服务有不同的定价和效果特点，您可以根据自己的需求选择合适的服务。由于视频生成涉及大量计算资源，大部分服务都需要付费使用。

## 四、配置指南

### 4.1 API Key 获取与配置

AIYOU 平台的核心能力来自于对各大 AI 服务商的 API 调用，因此正确配置各个平台的 API Key 是使用平台功能的前提。以下将详细介绍各个必需和可选的 API Key 的获取方法和配置步骤，帮助您快速完成配置准备工作。

Gemini API Key 是平台进行文本生成和图像理解的核心依赖。建议优先获取并配置此 API Key，因为它是角色设计、剧本生成等核心功能的底层支持。访问 Google AI Studio 网站（https://aistudio.google.com），使用您的 Google 账号登录，然后点击「Create API Key」按钮创建新的 API Key。创建完成后，复制生成的 API Key 并填入前端的 `.env.local` 文件中。

云屋 API（yunwu）是平台接入的国产大模型服务，在部分场景下可以提供比 Gemini 更好的中文支持效果。访问云屋 API 官网（https://yunwu.ai），注册账号并完成实名认证后，可以在用户中心获取 API Key。云屋 API 的配置位于前端设置面板中，无需修改代码文件，直接在应用内设置即可生效。配置后，平台会自动根据您的选择切换使用云wu 或 Gemini 进行文本生成。

视频生成相关的 API Key 包括 Sora2、Kling、Luma 和 Runway 等。这些服务的获取方式各有不同：Sora2 需要访问其官方网站申请内测资格；Kling 可以在可灵 AI 平台（https://klingai.com）申请；Luma 和 Runway 也分别需要访问各自的官方网站进行账号注册和 API Key 申请。需要注意的是，大部分视频生成服务目前仍处于测试阶段或需要付费使用。

### 4.2 环境变量详解

前端项目的环境变量主要存储在 `.env.local` 文件中，这个文件不会被 Git 版本控制系统跟踪，确保了敏感信息的安全性。以下是前端支持的所有环境变量及其说明，供您参考和配置。

`GEMINI_API_KEY` 是最重要的环境变量，用于配置 Google Gemini 模型的访问密钥。该 Key 用于几乎所有的文本生成功能，包括剧本创作、角色描述、分镜优化等。如果此变量未配置或配置错误，平台将无法正常运行文本生成功能。建议优先确保此变量配置正确。

后端服务使用独立的 `.env` 文件进行配置，主要包含腾讯云 COS 存储的配置信息。`OSS_BUCKET` 是 COS 存储桶的名称，格式通常为「您的名称-123456789」；`OSS_REGION` 是存储桶所在地域，如「ap-guangzhou」表示广州；`OSS_SECRET_ID` 和 `OSS_SECRET_KEY` 是您的腾讯云访问密钥，用于身份认证和数据访问。

### 4.3 模型配置

AIYOU 平台内置了智能的模型降级机制，当主模型不可用时会自动切换到备用模型，确保创作的连续性。模型配置信息存储在 `server/model-config.json` 文件中，包含各个模型提供商的基础 URL、模型名称、超时时间等参数。除非您需要对接自定义的 API 服务，否则一般情况下不需要修改此文件。

对于高级用户，可能需要根据实际需求调整模型配置参数。例如，您可以修改 `timeout` 参数来调整 API 请求的超时时间，这对于网络环境较差的用户可能需要延长超时时间以避免请求失败。您也可以修改 `maxTokens` 参数来调整生成内容的最大长度。如果您对配置进行了修改，需要重启后端服务以使更改生效。

平台还支持配置多个视频生成服务，配置信息同样位于 `model-config.json` 文件中。不同的视频服务有各自的特点和限制：Sora2 支持最长 10 秒的视频生成，Kling 支持 5-10 秒，Luma 和 Runway 的限制各有不同。您可以根据创作需求选择合适的服务，或者配置多个服务以实现故障转移。

## 五、操作流程

### 5.1 界面概览

AIYOU 采用现代化的节点式工作流设计，整个界面主要由画布区、节点面板、侧边栏和设置面板四个核心区域组成。理解各个区域的功能和相互关系，是高效使用平台的基础。以下将对各个界面区域进行详细介绍。

画布区是整个平台的核心工作区域，采用无限画布设计，您可以在这里自由拖拽、缩放和排列节点。画布支持多点触控操作，可以使用鼠标滚轮进行缩放，按住空格键或中键拖动可以平移画布视角。节点之间通过连线表示数据流转关系，连线会在节点准备好输出数据后自动建立，无需手动操作。

节点面板位于界面上方，包含所有可用的创作节点。这些节点按照功能分类，包括剧本创作、角色设计、分镜制作、视频生成等类别。点击节点面板中的节点图标，然后在画布上点击即可添加节点到工作流中。节点添加后会根据类型显示不同的配置选项，您可以在节点的属性面板中调整参数。

侧边栏位于界面左侧，提供对项目库、角色库和导出功能的快速访问。项目库显示所有保存的项目列表，可以快速切换或复制现有项目；角色库显示已创建的角色卡片，方便在不同项目中复用角色设计；导出功能支持将工作流导出为标准格式，便于项目备份或分享。

设置面板可以通过点击界面右上角的设置按钮打开，包含 API Key 配置、界面主题、语言设置等选项。建议在开始创作前先检查设置面板中的各项配置，确保 API Key 已正确配置且界面设置符合个人偏好。

### 5.2 创作工作流

使用 AIYOU 进行漫剧创作，遵循从剧本到视频的线性工作流程是最自然和高效的方式。虽然各个节点之间也支持任意的连接组合，但按照推荐的工作流进行创作可以获得最佳的体验和效果。以下是完整的创作工作流介绍。

第一步是使用「创意描述」节点输入您的故事想法。双击节点打开编辑器，输入几段关于故事主题、风格和主要情节的描述文字。AI 会根据您的描述扩展成一个完整的故事大纲。输入时尽量提供丰富的细节，包括故事背景、人物设定、情感基调等，这样 AI 生成的内容会更加符合您的预期。

第二步是使用「剧本大纲」节点生成剧本框架。连接创意描述节点的输出到大纲节点的输入，然后点击执行按钮。AI 会根据故事描述生成多集的剧本框架，每一集都有标题、简介和主要情节点。如果您需要调整集数或故事走向，可以在执行前修改节点的参数设置。

第三步是使用「剧本分集」节点生成分集剧本。连接大纲节点的输出到分集节点，选择您想要展开的某一集，执行后 AI 会生成完整的剧本内容，包括场景描写、人物对白、镜头指示等详细信息。生成的剧本支持进一步的人工编辑和调整。

第四步是创建角色设计。添加「角色设计」节点到画布，描述您想要的角色特征，包括性别、年龄、外貌特点、性格等。AI 会根据描述生成角色的三视图和九宫格表情包。生成的角色会自动添加到角色库中，方便后续在分镜节点中引用。

第五步是生成分镜脚本。连接剧本分集节点的输出到「分镜生成」节点，AI 会根据剧本内容自动生成专业的分镜脚本，包括景别、角度、运镜方式、时长等专业镜头语言。分镜脚本生成后，您可以在节点中预览和编辑每一镜的描述文字。

第六步是生成分镜图。连接分镜生成节点的输出到「分镜图设计」节点，选择合适的画面比例和布局，AI 会将文字描述转化为高清的分镜画面。支持生成九宫格或六宫格格式，方便预览连续镜头的效果。生成的分镜图可以直接用于视频参考或素材库存储。

第七步是视频生成。连接分镜图节点的输出到「分镜视频」节点，选择视频生成服务商（如 Sora2、Kling 等），设置生成参数后提交任务。视频生成是耗时最长的步骤，根据选择的模型不同，可能需要等待数分钟到数十分钟。生成完成后，视频会自动添加到项目中，您可以在线预览或下载到本地。

### 5.3 节点操作详解

平台中的每个节点都有相似的操作逻辑，包括添加、连接、配置、执行和删除等基本操作。熟练掌握这些操作技巧，可以大幅提升创作效率。以下详细介绍各个操作的细节和注意事项。

添加节点有两种方式：从节点面板拖拽节点到画布，或先在画布上点击定位，然后从面板选择节点类型添加。新添加的节点默认位于画布中心位置，您可以使用鼠标拖动来调整节点位置。节点支持批量选择和移动，适合在需要整理工作流布局时使用。

连接节点通过拖拽输出端口到目标节点的输入端口来完成。连接建立后，输出端口和输入端口之间会出现一条连线，表示数据流转关系。连线支持重路由以避免交叉混乱，拖拽连线的中间部分可以调整其路径。两个节点之间只能有一条连接，如果需要传递相同的数据给多个下游节点，可以使用分叉连接或复制节点。

节点配置通过点击节点打开属性面板进行。不同类型的节点有不同的配置参数，常见参数包括输入内容、生成参数、高级选项等。配置修改后，节点状态会从「已配置」变为「待执行」，需要点击执行按钮应用更改。部分参数支持实时预览，您可以在修改后立即看到效果变化。

执行节点会触发 AI 处理流程，根据节点的类型和输入内容生成相应的输出。执行过程中，节点会显示进度指示和日志信息，您可以点击展开查看详细的执行过程。如果执行失败，日志中会显示错误原因，根据错误信息进行排查和修正。成功执行后，输出数据会自动传递给下游节点。

删除节点通过选中节点后按 Delete 键或点击删除按钮完成。删除节点会同时删除与该节点相关的所有连接，但不会影响已经生成并保存到角色库或项目库中的内容。如果需要删除但保留输出数据，可以先将输出内容保存到库中，然后再删除节点。

### 5.4 数据管理

AIYOU 提供完善的数据管理功能，包括项目保存、数据导出和导入等。通过合理使用这些功能，可以有效保护您的创作成果，避免因意外导致数据丢失。以下介绍数据管理的具体方法和最佳实践。

项目保存是保护创作成果的基础操作。AIYOU 会自动保存项目的最近一次修改，但建议在完成重要阶段后手动保存项目。点击侧边栏中的保存按钮或使用快捷键 Ctrl+S（Windows）或 Cmd+S（macOS）可以保存当前项目。保存时会将整个工作流的状态保存到本地存储，包括所有节点、连接和配置信息。

数据导出功能支持将项目导出为标准格式的文件，便于备份或在另一台设备上继续工作。导出时可以选择导出完整项目或仅导出关键数据。导出文件为 JSON 格式，包含所有必要的信息。导入时只需将导出的文件拖入浏览器窗口，或使用导入功能选择文件，即可恢复项目状态。

角色库和素材库中的数据独立于项目存在，这意味着您在某个项目中创建的角色可以在其他项目中复用。角色库支持导出单个角色或全部角色的配置文件，方便在团队协作时共享角色设计资源。素材库中的图片和视频文件默认存储在云端 COS，删除本地项目不会影响云端存储的文件。

## 六、常见问题与解决

### 6.1 安装部署问题

在安装和部署 AIYOU 平台的过程中，可能会遇到各种技术问题。本节汇总了最常见的安装问题及其解决方法，帮助您快速解决部署过程中遇到的困难。如果以下方法仍无法解决您的问题，建议通过 GitHub Issues 提交问题反馈，或加入用户社区寻求帮助。

依赖安装失败是最常见的问题之一。如果在执行 `pnpm install` 时出现网络超时或下载失败的情况，首先检查您的网络连接是否正常，然后尝试切换 npm 镜像源。推荐使用国内镜像源，可以执行 `pnpm config set registry https://registry.npmmirror.com` 来设置镜像源。如果某个特定的依赖包安装失败，可以尝试删除 `node_modules` 文件夹和 `pnpm-lock.yaml` 文件后重新安装。

端口被占用是另一个常见问题。启动开发服务器时，如果提示端口已被占用，可以检查是哪个程序占用了该端口，然后结束该程序或修改平台使用的端口号。前端默认使用 5173 端口，可以通过在 `.env.local` 中设置 `VITE_DEV_PORT=新端口号` 来修改；后端默认使用 3001 端口，可以通过 `server/.env` 中的 `PORT` 变量修改。

Node.js 版本不兼容可能导致各种运行时错误。建议使用 Node.js 18.x 或 20.x LTS 版本，可以使用 `nvm use 18` 或 `nvm use 20` 来切换版本。如果使用 nvm 管理 Node.js 版本，确保在正确的版本下执行安装和启动命令。可以通过 `node -v` 命令确认当前使用的 Node.js 版本。

### 6.2 功能使用问题

即使成功部署了平台，在实际使用过程中仍可能遇到各种功能相关的问题。本节针对用户反馈最多的功能使用问题提供解决方案，帮助您更好地使用平台进行创作。

API 请求失败是影响最大的问题类型。遇到此问题时，首先检查 API Key 是否正确配置，确认没有多余的空格或换行符。然后检查网络连接是否能够访问相应的 API 服务，某些地区的网络可能无法直连 Google 等境外服务。最后检查 API 配额是否耗尽，部分服务有每日调用次数限制。

视频生成失败可能由多种原因导致。首先确认您已配置有效的视频生成服务 API Key，且配额充足。其次检查生成参数是否在服务支持范围内，例如视频时长、画面比例等限制。视频生成是耗时操作，如果等待时间过长显示超时，可以尝试降低生成分辨率或缩短视频时长。如果问题持续，可能是服务端临时不可用，建议稍后重试。

节点连接后不执行通常是因为上游节点没有输出数据。检查上游节点是否已成功执行，可以通过查看节点状态指示器判断。确认输入数据格式符合节点要求，某些节点对输入内容有最低字数要求。检查是否存在循环引用或无效连接，这类情况会导致数据无法正确传递。

### 6.3 性能优化建议

AIYOU 平台涉及大量的 AI API 调用和媒体文件处理，这些操作可能对系统资源造成较大压力。以下是一些性能优化建议，可以帮助您获得更流畅的使用体验。

浏览器性能优化方面，建议使用 Chrome 或 Edge 等性能较好的浏览器，并确保浏览器已启用硬件加速。在系统设置中关闭不必要的后台程序，释放内存资源给浏览器使用。定期清理浏览器缓存，避免缓存文件过大影响性能。如果您的设备内存较小，建议只保留必要的浏览器标签页。

API 调用优化方面，可以通过调整参数减少不必要的 API 消耗。例如，在角色设计时选择较小的图像尺寸，在分镜图生成时使用合适的分辨率而非最高分辨率。对于不需要实时响应的任务，可以使用较低优先级的队列进行处理。合理利用缓存机制，避免对相同内容重复生成。

存储空间管理方面，定期清理不再需要的项目和素材，删除无用的测试视频文件。将不常用的项目导出为备份文件后从本地删除，释放存储空间。如果使用腾讯云 COS，注意监控存储费用，在必要时清理过期文件。

## 七、注意事项

### 7.1 安全注意事项

使用 AIYOU 平台时，需要注意以下安全相关的事项，以保护您的账户安全、数据安全和隐私安全。正确理解和遵守这些注意事项，可以避免不必要的安全风险和损失。

API Key 是您使用各 AI 服务的身份凭证，必须妥善保管。不要将包含真实 API Key 的配置文件分享给他人或上传到公开的代码仓库。平台的前端代码中不包含任何 API Key，所有 Key 都存储在您本地的环境变量文件中。如果您怀疑 Key 已经泄露，应立即到相应的服务商平台撤销并重新生成。

数据安全方面，平台生成的所有内容默认保存在您的本地设备上，除非您主动上传到云存储服务。建议定期备份重要的项目文件和创作素材，特别是使用 COS 云存储时，确保您已了解数据存储和传输的安全性设置。

网络使用安全方面，平台只与官方声明的服务端点进行通信，不会将您的数据发送给第三方服务。在使用过程中，避免点击任何来源不明的链接，防止钓鱼攻击。如果您发现任何可疑行为，应立即停止使用并检查系统。

### 7.2 使用限制与合规

AIYOU 平台集成了多个第三方 AI 服务，使用这些服务需要遵守各服务提供商的使用条款和限制。以下是一些重要的合规注意事项，帮助您合法合规地使用平台进行创作。

API 调用限制方面，各 AI 服务提供商都有不同的调用频率和数量限制。这些限制根据您的账户类型和服务级别有所不同，超出限制可能导致请求被拒绝或产生额外费用。请在使用前了解并遵守各服务的使用条款，避免因违规使用导致账户被封禁。

内容合规方面，您使用平台生成的内容需要符合各服务提供商的内容政策。严禁生成违法、侵权、欺诈或其他不当内容。平台会记录 API 调用的日志信息，包括请求内容和服务响应，这些日志仅用于问题排查和服务改进，不会用于其他目的。

版权声明方面，使用 AIYOU 生成的创作成果，其知识产权归属遵循各 AI 服务提供商的政策和您所在国家或地区的法律规定。一般情况下，AI 生成内容的版权归属于使用者，但某些服务可能有特殊的条款限制。建议在使用前仔细阅读各服务的服务条款，了解您的权利和限制。

### 7.3 最佳实践

以下是长期使用 AIYOU 平台的最佳实践总结，遵循这些实践可以帮助您更高效、更安全地使用平台进行创作。这些经验来自于开发团队和社区用户的反馈，具有很高的参考价值。

项目组织方面，建议按照创作阶段划分项目，每个阶段使用独立的项目文件保存。这样既方便回溯和修改，也便于在不同项目间复用角色和素材。定期整理角色库和素材库，删除不再使用的内容，保持库的整洁有序。

工作流优化方面，养成良好的工作习惯，在完成每个创作阶段后保存项目。对于复杂的创作任务，可以先将想法通过「创意描述」节点进行扩展，然后再进入正式的剧本创作流程。合理利用节点参数的默认值，减少重复配置，提高工作效率。

学习提升方面，平台内置了 20 多套精心调教的提示词模板，这些都是经过反复优化的创作模板。新手用户可以先从模板开始，了解如何撰写有效的提示词，然后逐步发展出自己的创作风格。遇到问题时善用搜索功能查看文档和社区讨论，大多数问题都有现成的解决方案。
